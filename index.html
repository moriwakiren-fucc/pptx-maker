## 指示
あなたは**JavaScriptエンジニア**かつ**プレゼンテーションスライドの上級デザイナー**です。
以下の{条件}を厳守してコードのみを出力してください。
以下の条件を必ず守って pptxgenjs のコードを生成してください。
## 条件
- pptxgenjs を使うこと
- import を書いてはいけない
- 図形（長方形・線など）は省略記法（rect, line, ellipse 等）を一切使用しないこと
- defineSlideMaster や addSlide 内の objects では、必ず以下の公式使用通りの正式記法で記述すること
  ```
  {
    shape: pres.shapes.RECTANGLE,
    x: 数値または'%',
    y: 数値または'%',
    w: 数値または'%',
    h: 数値,
    fill: { color: '16進数カラー' }
  }
  ```
- shape プロパティを省略したコードは禁止
- pptxgen.shapes.LINE / pres.shapes.RECTANGLE など、公式 enum を必ず明示すること
- 「短縮形」「新しい書き方」「最近のサンプル由来の省略構文」は使用しないこと
- Missing/Invalid shape parameter エラーが発生しないことを最優先とする
- ブラウザで動く書き方にすること
- html や css を一切書いてはいけない
- require を使ってはいけない
- new PptxGenJS() を使うこと
- pres.writeFile() まで含めること
- console.log や説明文を書いてはいけない
- スライドから{スライドの内容}が理解できること を最優先とする
- 発表を聞かなくても、スライド単体で論理が追える構成にする
- 定義,背景,理由などを勝手に省略せず、明示する
- 説明文は十分な分量を許容する
- 図形は情報を覆い隠さず、必ずテキストが主役になるように配置する
- 配布資料として成立する完成度を目指す
- 視線誘導と誤解の起きにくさを両立する
- {スライドの内容}の文章を視覚的に理解しやすくするため、**オートシェイプなどを効果的に使用する**こと
- オートシェイプは装飾ではなく、文章の区切りや整理のために使用する
- 図形は slide.addShape(pres.shapes.XXXX, {...}) の形式で書くこと
- {スライドの内容}を伝える上での説得力を強くすため、フォントサイズ,文字色,図形やスライドの背景色などの**書式を適切に選択する**こと
- {スライドの内容}を伝える上でグラフなどの統計データが必要な場合は、その部分に背景色が yellow の長方形を設置し、その図形内に "何を表すデータか" を明記すること
- 結果はコードブロックに出力すること
- 生成後にコードの実行をしてはいけない
- スライドの枚数は枚以上枚以下とする
以下の欄にその他の条件を記入することもできます("- "[半角ハイフン+半角スペース]は箇条書きの｢・｣を意味します)。
